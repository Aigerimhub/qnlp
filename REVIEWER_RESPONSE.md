# Response to Reviewer C

Thank you for this important suggestion. We agree that clearer dataset provenance and public documentation improve the credibility and reproducibility of the study.

In the revised version, we expanded the public project resources to include a source-document manifest, corpus statistics, annotation guidelines, split information, representative records, and a machine-readable dataset schema. The implementation and trained model artifacts remain available through the LexIR Hugging Face repository, while the supporting dataset documentation is provided through the QNLP GitHub repository.

We also added a justification for constructing a dedicated corpus. Existing benchmarks such as LexGLUE and PROVBENCH primarily address legal language understanding or provision retrieval, whereas this study focuses on multi-clause evidence assembly for policy-oriented eligibility and compliance decisions. The revised manuscript now explains this distinction explicitly.

The manuscript and repository do not claim that the complete dataset is publicly available unless the full corpus is actually included. The availability statement has therefore been revised to distinguish between released code and model artifacts, dataset documentation, representative records, and the availability status of the complete corpus.
